### 1. 核心角色介绍

#### **角色 A：GOST (GO Simple Tunnel)**

- **它的身份**：**“内网代购员”**。
- **它的本质**：一个功能强大的代理服务软件。
- **在这个方案中的作用**：
    - 它运行在你实验室电脑的 `1080` 端口上。
    - 它的工作很简单：谁连上它，告诉它想访问什么网页（比如浙大内网），它就利用实验室电脑的校园网权限，去把那个网页的数据取回来，再发给请求者。
    - **为什么需要它？** 因为 FRP 只负责修路（传输数据），它不负责“上网”。如果没有 GOST，数据传到了实验室电脑却没人处理，就像你把信寄到了，但没人帮你读信办事。

#### **角色 B：FRP (Fast Reverse Proxy)**

- **它的身份**：**“地道挖掘队”**。
- **它的本质**：内网穿透工具。
- **在这个方案中的作用**：
    - **FRP Server (frps)**：在你新加坡服务器上挖了一个洞口（端口 `10080`）。
    - **FRP Client (frpc)**：在你实验室电脑上，把这个洞口和新加坡连起来。
    - **核心逻辑**：因为实验室电脑在内网，没有公网 IP，外面的数据进不来（防火墙挡着）。FRP 的巧妙之处在于，让**实验室电脑主动向新加坡服务器发起连接**（内网出去通常是不拦的），建立一条长连接（地道）。一旦地道通了，外面的数据就可以顺着这条地道反向流进来。

### 2. 详细步骤深度解析

现在我们把你引用的操作步骤逐行拆解，看看它们到底在干什么：

#### **步骤 1：下载工具**

- 这没啥好说的，就是准备作案工具：一个负责转发请求（GOST），一个负责打通网络（FRP）。

#### **步骤 2 - A：配置 `frpc.toml`**

这是告诉“挖掘队”怎么干活：

toml

```
serverAddr = "你的新加坡服务器IP"  # 告诉本地电脑，地道出口在哪
serverPort = 7000                  # 这是和服务器握手的“联络端口”

[[proxies]]
name = "campus-proxy"
type = "tcp"                       # 只要是 TCP 数据包，不管内容是啥，原封不动地传
localIp = "127.0.0.1"
localPort = 1080                   # 【关键点】把从地道传进来的数据，交给本地谁处理？交给 1080 端口（也就是 GOST）
remotePort = 10080                 # 【关键点】在新加坡服务器上开哪个门？开 10080
```

- **翻译**：只要有人访问新加坡服务器的 `10080` 端口，FRP 就会把数据包通过地道搬运到实验室电脑，然后扔给实验室电脑的 `1080` 端口。

#### **步骤 2 - B：启动脚本 `start.bat`**

这个脚本同时启动了两个服务：

**第一行命令：**  
`start /b gost-windows-amd64.exe -L socks5://:1080`

- **含义**：启动 GOST，让它监听本地的 `1080` 端口，协议是 `SOCKS5`。
- **潜台词**：“我是 GOST，我就站在实验室电脑的 1080 门口，谁给我发请求，我就帮谁去跑腿访问校园网。”

**第二行命令：**  
`start /b frpc.exe -c frpc.toml`

- **含义**：启动 FRP 客户端，读取刚才的配置文件。
- **潜台词**：“我是 FRP，我现在要连上新加坡服务器，建立地道。一旦那边有数据过来，我就全部转交给刚才站在 1080 门口的 GOST。”

### 3. 完整的数据流向（Traffic Flow）

当你坐在校外的咖啡馆，在浏览器输入 `cc98.org` 时，发生了什么？

1. **你的电脑 (Clash)**：发现 `cc98.org` 匹配了规则，于是把请求打包，发送给 `新加坡服务器IP:10080`。
2. **新加坡服务器 (frps)**：监听着 `10080` 端口，收到请求。它知道这个端口对应着你实验室的那台电脑，于是把数据包塞进“FRP 隧道”。
3. **穿过互联网**：数据包通过加密隧道传输。
4. **实验室电脑 (frpc)**：从隧道里收到数据包，根据配置，它把数据包扔给了本地的 `127.0.0.1:1080`。
5. **实验室电脑 (GOST)**：它正监听着 `1080`，收到了数据包。它拆开一看：“哦，老板要访问 `cc98.org`”。
6. **实验室电脑 (校园网)**：GOST 代表你去访问 `cc98.org`。因为这台电脑就在校内，所以访问毫无阻碍。
7. **回程**：CC98 的网页内容原路返回：**CC98 -> GOST -> frpc -> 隧道 -> frps -> 你的电脑**。

### 总结

- **GOST** 是**“嘴巴”**，负责帮你在内网说话、办事。
- **FRP** 是**“传声筒”**，负责把你远在天边的指令传到内网。
- **两者缺一不可**：只有 FRP，你连上了电脑但干不了事；只有 GOST，你能在内网办事但你在外面连不上它。
